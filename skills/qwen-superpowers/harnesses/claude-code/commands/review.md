Apply the Requesting Code Review procedure (canonical: skills/requesting-code-review.md).

1. Verify first (/verify) — never request review on red or unverified work.
2. Summarize in 2–4 lines: what changed and why; link the spec/plan.
3. Show the scoped diff (one concern).
4. Call out risky / uncertain / out-of-scope parts.
5. State how you tested it (commands + output).

Split large diffs; don't bundle unrelated refactors.
