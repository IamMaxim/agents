---
description: Turn an approved design/spec into a numbered plan of small, checkable steps.
argument-hint: <spec or design>
---

Apply the Writing Plans procedure (canonical: skills/writing-plans/SKILL.md).

1. List ordered steps; each changes ONE file or concern.
2. Give each step a concrete check (the command/observation that proves it works). No check = not done.
3. Order so every step leaves the tree green and the app runnable.
4. Save the plan to docs/plans/<topic>.md so /execute-plan can follow it.

Make steps smaller than feels necessary; each must be independently revertable. Flag risky steps
for a worktree (skills/using-git-worktrees/SKILL.md).

Spec / design: $ARGUMENTS
